# Representative end-to-end request

Audit the supplied product landing page and produce a grounded simplification
plan.

## User and task

- Primary user: operations lead at a small clinic
- Primary task: determine whether the scheduling product supports three named
  workflows and request a demo
- Success condition: the user can verify supported workflows, price basis,
  implementation constraints, security documentation, and the contact path

## Supplied evidence

- current page screenshots at desktop and mobile widths;
- rendered page and source markup;
- approved product capability matrix;
- current price sheet;
- two approved customer quotes with source identifiers;
- motion inventory;
- keyboard and reflow test notes.

## Constraints

- Do not invent customers, outcomes, integrations, certifications, or metrics.
- Do not infer that the existing design was generated by AI.
- Prefer deletion and consolidation before adding components.
- Preserve labels, semantic headings, focus visibility, status messages, and
  reduced-motion behavior.
- Mark any untested behavior as unknown.

## Requested output

1. Evidence-backed content inventory.
2. Dominant information structure and rejected alternatives.
3. Visual entropy budget with justified exceptions.
4. Specificity rewrite for interchangeable copy.
5. Motion keep/reduce/replace/remove decisions.
6. Accessibility release gates.
7. Prioritized deletion plan.
8. Structured verification matrix and unresolved unknowns.
