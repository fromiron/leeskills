# Anti-AI-Slop Orchestrator

Coordinate the smallest set of focused skills needed to ground, simplify, and
verify a digital artifact.

## Operating principle

Use the smallest coherent visual and verbal system that preserves:

1. the user's primary task;
2. the artifact's verified content;
3. necessary functionality;
4. accessibility;
5. the author's or product's specific identity.

Do not infer that an artifact was generated by AI from style alone. Report
observable slop signals, not authorship.

## Required inputs

Use what the user supplied:

- artifact or source files;
- screenshot, URL, markup, component tree, or design tokens;
- intended user and primary task;
- real copy, claims, dates, prices, outcomes, and constraints;
- target platform and release stage.

When critical evidence is absent, record it as unknown. Do not invent it.

## Evidence labels

Label every material statement:

- **Observed** — directly present in supplied material.
- **Measured** — produced by a deterministic check.
- **Inferred** — reasoned from evidence; state that it is an inference.
- **Unknown** — not verifiable from the supplied material.

Never convert inferred or unknown material into a factual claim.

## Decision gates

Do not issue a passing result while any of these project hard failures remain:

- fabricated customer, metric, quote, award, result, capability, or case study;
- mock or generated UI presented as proof of a real product capability;
- removal of keyboard focus, form labels, errors, text alternatives, or
  semantic structure for visual cleanliness;
- scroll hijacking without an equivalent user-controlled mode;
- essential information available only through motion, color, hover, or
  pointer-only interaction;
- nonessential interaction motion without a reduction or disable path;
- content that cannot reflow without loss at the target accessibility baseline;
- unsupported claims presented with certainty.

A hard failure is a project release gate. It is not automatically a legal
conclusion or a complete WCAG conformance determination.

## Deliverable

Use [assets/full-report-template.md](assets/full-report-template.md).

Include:

1. scope and supplied evidence;
2. primary user and task;
3. observed signals;
4. selected structure;
5. content gaps and prohibited inventions;
6. visual budget and justified exceptions;
7. copy changes;
8. motion decisions;
9. accessibility findings;
10. deletion plan ordered by impact and reversibility;
11. verification status;
12. unresolved unknowns.

## Completion rule

Do not optimize for emptiness. Stop pruning when the next removal would reduce
task completion, comprehension, evidence, identity, or accessibility.
