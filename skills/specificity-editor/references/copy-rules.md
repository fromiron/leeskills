# Copy rules

## Substitution test

Temporarily replace the organization, product, or author name with an unrelated
one. If the sentence remains equally plausible, it probably lacks identity or
evidence.

This is a diagnostic, not a ban on common words.

## Claim classes

### Identity

State who or what the artifact represents.

### Offer

State the actual product, service, role, work, or information available.

### Mechanism

State how it works when that matters to the decision.

### Evidence

Use attributable work, behavior, data, customers, dates, or outcomes.

### Constraint

State price, eligibility, platform, scope, availability, risk, or limitation.

### Action

Use a verb that matches the destination or operation.

## Rewrite examples

Generic:

```text
We create seamless experiences that empower the future.
```

More specific:

```text
We design and build scheduling tools for independent medical clinics.
```

The second sentence is only valid when those facts are supplied.

Generic:

```text
Unlock powerful insights.
```

Possible grounded versions:

```text
Compare weekly activation and retention by acquisition channel.
```

```text
Export transaction summaries as CSV.
```

Choose only the version supported by the actual product.

## Evidence-sensitive language

Use:

- `shows`, `supports`, or `reported` when evidence is available;
- `designed to` for an intended effect not yet measured;
- `may` or `can` for a contextual possibility;
- `planned` or `proposed` for roadmap items.

Do not use qualification as camouflage for a claim with no useful basis.

## Numbers

A number increases specificity only when it is correct, current, and
attributable. Dates, prices, percentages, time savings, customer counts, and
performance claims require source review.

## Voice

Preserve distinctive vocabulary, rhythm, humor, and domain terminology when
they are intentional. Remove generic filler, not authorial character.
